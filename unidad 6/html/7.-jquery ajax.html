JQuery AJAX

Remember: Ajax is not a new technology, in fact, Ajax is not even really a technology at all. Ajax is just a term to describe the process of exchanging data from a web server asynchronously through JavaScript, without a page refresh.

How it works: your code sends a request to a URL, and when the request completes, a callback function can be triggered to handle the response. Further, since the request is asynchronous, the rest of your code continues to execute while the request is being processed.

Ajax request can be made only to the files that exist on the same web server that servers the page from which the Ajax request is sent, not to external or remote servers for security reasons. This is called same-origin policy.

load (URL, data, successCallback) -> loads an HTML page or framgment into a container

get (URL, data, successCallback, dataType) -> gets information from a server

post (URL, data, successCallback, dataType) -> sends information to a server

getJSON (URL, data, successCallback)-> getJSON Load JSON-encoded data from the server using a GET HTTP request (similar to get)

only URL is mandatory

  • URL
  • data (optional): set of query string (i.e. key/value pairs) that is sent to the web server along with the request
  • successCallback (optional). It has three parameters
    • responseTxt — Contains the resulting content if the request succeeds.
    • statusTxt — Contains the status of the request such as success or error.
    • jqXHR — Contains the XMLHttpRequest object.
  • dataType (optional) is the returned type. If not specified, it is got based on MIME type of response
